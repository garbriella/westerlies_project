# WesterlyScope | 상층 편서풍과 지상 서풍대 분석기

NOAA 고층·지상 관측과 재분석 자료를 이용하여 상층 편서풍의 연직 강화와 중위도 지상 서풍대의 유지 원리를 분석하는 Python·Streamlit 탐구 애플리케이션입니다.

최종 탐구 주제는 **“상층 편서풍과 지상 서풍대는 같은 원인으로 형성되는가: 열풍 관계와 중위도 에디의 운동량 수송을 중심으로”**입니다.

## 탐구동기

위도에 따른 태양에너지 수수 차이가 적도와 극 사이의 기온 경도를 만들고, 이것이 열풍 관계를 통해 고도가 높아질수록 편서풍이 강해지는 현상과 연결됨을 학습하였다. 그러나 마찰이 크게 작용하는 지표에서도 중위도의 장기 평균 바람이 서풍 성분을 유지하는 이유는 남북 기온 차만으로 충분히 설명하기 어렵다고 판단하였다. 이에 NOAA의 고층·지상 관측과 재분석 자료를 이용하여 상층 편서풍의 연직 강화와 지상 서풍대의 유지 과정이 어떻게 다르고 연결되는지 분석하고자 한다.

## 연구 질문

1. 중위도에서는 고도가 높아질수록 서풍 성분이 실제로 강해지는가?
2. 남북 기온 차로 계산한 열풍 관계의 예측값은 관측된 연직 서풍 시어와 일치하는가?
3. 지표의 서풍은 상층 서풍보다 약하고 변동성이 큰가?
4. 중위도 에디의 동서 방향 운동량 수송은 평균 서풍 유지에 어떤 방향으로 작용하는가?
5. 상층 편서풍과 지상 서풍대를 같은 형성 원리로 설명할 수 있는가?

## 이론적 배경

### 바람 성분과 편서풍

기상학적 풍향은 바람이 **불어오는 방향**이다. 앱은 MetPy의 `wind_components`를 우선 사용해 풍향·풍속을 동서 성분 `u`와 남북 성분 `v`로 변환한다.

- `u > 0`: 공기가 동쪽으로 이동하는 서풍 성분
- `u < 0`: 공기가 서쪽으로 이동하는 동풍 성분
- `v > 0`: 공기가 북쪽으로 이동하는 남풍 성분
- `v < 0`: 공기가 남쪽으로 이동하는 북풍 성분

따라서 “서풍 성분이 양수”는 바람이 서쪽에서 불어 동쪽으로 이동한다는 뜻이다. 순간 풍향 한 번이 아니라 장기 평균 `u`와 `u > 0` 발생 비율로 서풍대를 진단한다.

### 지균풍과 열풍 관계

지균풍은 수평 기압경도력과 코리올리 힘이 균형을 이룬 이상화된 바람이다. 열풍은 새로운 독립 바람이 아니라 두 기압면 사이의 **지균풍 차이**다. 두 관측소의 층평균 기온으로 동서풍 시어를 다음처럼 근사한다.

```text
Δu_tw = -(R_d / f) × ln(p_lower / p_upper) × (∂T̄ / ∂y)
Δu_obs = u_upper - u_lower
```

`R_d`는 건조공기 기체상수, `f`는 중간 위도의 코리올리 매개변수다. 북쪽·남쪽 관측소의 평균 관측 시어와 예측값을 비교해 상관계수, 회귀 기울기, RMSE와 부호 일치율을 계산한다.

### 지표 서풍과 에디 운동량 수송

지표에서는 마찰, 지형, 해륙풍, 계절풍과 이동성 고·저기압 때문에 순간 풍향 변동이 크다. 지표–상층 차이는 마찰 영향을 **간접 진단**하는 자료이며 마찰력의 크기를 직접 산출하지 않는다.

재분석 일별 바람에서 시간평균으로부터의 편차를 계산한다.

```text
u' = u - time_mean(u)
v' = v - time_mean(v)
eddy flux = time_and_zonal_mean(u'v')
A_eddy = -1 / (a cos²φ) × ∂[cos²φ × overline(u'v')] / ∂φ
```

양의 `A_eddy`는 평균 동쪽 방향 흐름을 강화하는 방향의 진단값이다. 이 값은 전체 대기 운동량 방정식이 아니며 지상 마찰, 산악 토크, 평균 자오면 순환 등 다른 항을 포함하지 않는다.

## 사용 데이터와 NOAA 출처

### NOAA IGRA 2.2

라디오존데의 기압별 기온, 고도, 풍향과 풍속을 사용한다. 기본 관측소는 포항(`KSM00047138`)이며, 앱이 [공식 관측소 목록](https://www.ncei.noaa.gov/data/integrated-global-radiosonde-archive/doc/igra2-station-list.txt)에서 국가 코드 `KS`인 대한민국 관측소를 추출한다.

- [전체기간 자료](https://www.ncei.noaa.gov/data/integrated-global-radiosonde-archive/access/data-por/)
- [최근 자료](https://www.ncei.noaa.gov/data/integrated-global-radiosonde-archive/access/data-y2d/)
- [IGRA 2.2 고정폭 형식](https://www.ncei.noaa.gov/data/integrated-global-radiosonde-archive/doc/igra2-data-format.txt)

파일명이나 시작 연도를 고정하지 않고 디렉터리에서 관측소 ID가 포함된 파일을 탐색한다. 전체기간 자료를 먼저 사용하고 실패하면 최근 자료 후보를 시도한다. 기압은 Pa→hPa, 기온과 풍속은 0.1 단위 스케일을 적용하며 `-8888`, `-9999`를 결측으로 처리한다.

### NOAA Integrated Surface Database

[ISD 안내](https://www.ncei.noaa.gov/products/land-based-station/integrated-surface-database)와 [관측소 이력](https://www.ncei.noaa.gov/pub/data/noaa/isd-history.csv)을 사용한다. IGRA 위치에서 대한민국 후보를 찾고 이름 일치와 대권거리로 정렬한다. 기본 허용 거리는 30 km이며 사용자가 후보를 직접 바꿀 수 있다.

선택 기간에 해당하는 연도별 Global Hourly CSV만 요청한다. `USAF`와 `WBAN`은 문자열로 유지한다. `WND`, `TMP`, `DEW`, `SLP`의 누락 코드와 좋지 않은 품질 플래그는 `NaN`으로 바꾼다.

### NOAA NCEP/DOE Reanalysis II

- [자료 안내](https://psl.noaa.gov/data/gridded/data.ncep.reanalysis2.html)
- [일별 기압면 THREDDS 카탈로그](https://psl.noaa.gov/thredds/catalog/Datasets/ncep.reanalysis2/Dailies/pressure/catalog.html)

카탈로그에서 `uwnd`와 `vwnd`가 모두 존재하는 연도를 찾고, 현재 진행 중인 연도를 제외한 가장 최근 연도를 기본값으로 고른다. 사용자가 실행 버튼을 누른 경우에만 OPeNDAP을 열며 계절, 20–70°N(변경 가능), 850/300 hPa를 먼저 선택하고 모든 경도만 유지한다. 전체 netCDF 파일을 내려받지 않는다.

## 분석 방법

### 표준기압면 보간과 품질관리

표준면은 1000, 925, 850, 700, 500, 300, 250, 200 hPa다. 정확한 레벨이 없으면 로그 기압 좌표에서 선형 보간한다.

- 관측 범위 밖으로 외삽하지 않는다.
- 양옆 레벨 기압 차가 250 hPa를 넘으면 보간하지 않는다.
- 풍향 대신 `u`, `v`를 각각 보간한 뒤 풍향·풍속을 복원한다.
- 실제값과 보간값을 `is_interpolated`로 구분한다.
- 유효값이 부족하면 `NaN`을 유지한다.

### 지상·상층 시간 대응

UTC를 기준으로 각 sounding 전후 90분(사용자 설정 30–180분) 안의 가장 가까운 유효 ISD 관측을 선택한다. 중복 시각이면 좋은 품질 자료를 먼저 사용한다. 허용 범위 안에 관측이 없으면 강제로 대응하지 않는다. 화면과 다운로드 표에는 UTC, KST(UTC+9) 및 시간 차를 표시한다.

### 계절

겨울은 12–2월, 봄은 3–5월, 여름은 6–8월, 가을은 9–11월이다. 12월을 다음 해 겨울로 묶는 `season_year`를 별도로 계산한다. 재분석 겨울도 전년 12월과 선택 연도 1–2월을 결합한다.

## 앱 기능

1. **연구 개요**: 탐구동기, 연구 질문, u/v와 열풍·에디 개념 도식
2. **고도별 서풍 구조**: 평균 u/v, 풍속, 변동성, 서풍 비율, 계절·개별 sounding 프로파일
3. **지상과 상층 비교**: ISD–IGRA 시간 대응, 상관·차이·변동성, 산점도·풍향 빈도
4. **열풍 관계 검증**: 북·남 관측소 지도, 예측/관측 시어, 계절 비교와 자동 해석
5. **에디 운동량 수송**: 평균 u, `u'v'`, 구면 수렴, 850/300 hPa 비교
6. **종합 분석**: 세 증거의 조건부 해석, CSV 6종과 Markdown 요약 다운로드
7. **자료와 방법**: 이론, 품질관리, 자료 출처와 과학적 한계

페이지 하단 댓글은 `st.session_state`에만 저장되며 영구 저장되지 않는다. 네트워크 또는 특정 모듈 오류가 다른 관측 탭 전체를 중단하지 않도록 기능별 예외 처리를 적용했다.

## 로컬 실행

Python 3.12가 필요하다.

```bash
python3.12 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
pytest -q
streamlit run main.py
```

Windows PowerShell에서는 가상환경 활성화 명령을 `.venv\Scripts\Activate.ps1`로 바꾼다.

## Streamlit Community Cloud 배포

배포 설정은 다음 값을 사용한다.

- Repository: `westerlies_project`
- Branch: `main`
- Main file path: `main.py`
- Python version: **3.12**

`.python-version`에도 `3.12`가 지정되어 있지만, Streamlit Community Cloud의 **Advanced settings에서 Python 3.12를 직접 선택하는 것이 가장 확실하다.** Python 3.14로 배포하면 일부 오래된 과학 패키지에서 호환성 문제가 생길 수 있으므로 이 프로젝트는 Python 3.12를 기준으로 한다.

배포 순서는 다음과 같다.

1. 변경 사항을 GitHub의 `main` 브랜치에 push한다.
2. Streamlit Community Cloud에서 새 앱을 생성한다.
3. 저장소로 `westerlies_project`를 선택한다.
4. 실행 파일로 `main.py`를 선택한다.
5. Advanced settings에서 Python 3.12를 선택한다.
6. Deploy를 실행한다.
7. 배포 후 앱 로그에서 import 오류와 패키지 설치 오류를 확인한다.
8. 코드를 수정하여 GitHub에 다시 push하면 같은 URL의 앱에 자동 반영된다.
9. `requirements.txt`가 바뀐 배포는 패키지 재설치 때문에 업데이트 시간이 더 걸릴 수 있다.
10. 이전 빌드 캐시 때문에 오류가 계속되면 **Manage app → Reboot app**을 실행한다.

앱은 NOAA 공개자료만 사용하며 API 키나 secrets가 필요하지 않다. 대용량 NOAA 원자료, CSV, ZIP, netCDF는 `.gitignore` 대상이다.

## 과학적 한계

- 단일 관측소나 두 관측소 결과로 전 지구 중위도 순환을 확정할 수 없다.
- 두 열풍 관측소는 정확히 같은 경도선에 있지 않고 연속 기온장을 대표하지 못한다.
- 관측 바람에는 비지균 성분, 지형, 국지순환과 시각·고도 불일치가 포함된다.
- 지표와 상층의 차이만으로 마찰력을 직접 산출할 수 없다.
- 에디 수렴은 전체 운동량 수지의 일부이며 다른 역학 항을 생략한다.
- 재분석은 관측과 수치모형이 결합된 자료로 모형 및 동화 체계의 영향을 받는다.
- 특정 한 해의 재분석 결과는 장기 기후학적 특성과 같지 않을 수 있다.
- 상관관계가 곧 인과관계는 아니다.

따라서 상층 편서풍과 지상 서풍대는 서로 관련되어 있지만 동일한 단일 과정의 결과라고 보지 않는다. 상층에서는 남북 기온 경도와 열풍 관계가 연직 강화와 연결되고, 지표에서는 마찰·이동성 교란·에디 운동량 수송을 포함한 복합 운동량 수지가 중요하다. 선택 자료가 이를 충분히 지지하지 않으면 앱은 “이론적으로 예상되지만 선택 자료에서는 명확히 확인되지 않음”으로 제한해 표시한다.

## 오류 해결

- **IGRA 파일 없음**: 관측소 자료 종료연도와 선택 기간을 확인한다. 앱은 전체기간 뒤 최근자료를 시도한다.
- **ISD 대응 없음**: 후보 거리, 관측소 운영기간, 허용 범위(최대 180분)를 확인한다.
- **열풍 유효쌍 없음**: 북쪽 위도가 더 높은지, 두 기압면의 기온·u가 모두 유효한지 확인한다.
- **OPeNDAP/netCDF 오류**: NOAA 서버 상태를 확인하고 다른 완전 연도나 더 짧은 계절을 시도한다. `netCDF4`와 `cftime` 설치도 확인한다.
- **한글 표시 문제**: 그래프는 브라우저 시스템 글꼴로 렌더링하며 한글 전용 Matplotlib 패키지를 요구하지 않는다.
- **캐시 갱신**: 사이드바의 **자료 새로고침**을 눌러 Streamlit 데이터 캐시와 현재 분석 자료를 비운다.
